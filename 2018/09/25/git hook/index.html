git hook

有时候, 在使用git提交代码的时候, 需要先用语法检查工具来检查项目中是否有不符合规范的语法, 如果有, 就不能提交代码

所以就要 git hook
比如commit 前执行 eslint的检测操作, 如果eslint的语法检查没有校验通过则不能进行提交

要安装 husky 依赖, 注意安装依赖的时候 目录一定是要初始化好的 git 仓库, 不然此依赖找不到对应的git 进行hook
